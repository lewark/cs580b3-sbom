# Logs Parser and Analyzer

This directory contains scripts and directories for parsing and formatting the raw JSON responses generated by the LLM agents during the evaluation process.

## `parser.py`

The `parser.py` script is used to traverse through agent-generated log files, extract the final structured output from the agent's response, and format it cleanly into the `parsed-logs` directory. It supports both single-file parsing and recursive directory traversal.

### Directory Structure

The structure of this directory looks generally like this:
- `<target_app>/`: (e.g. `tomcat/`) Contains raw logs nested inside iteration subfolders (like `iteration1/`, `iteration2/`).
- `parsed-logs/`: The directory where the cleaned logs will be generated. The script will automatically mirror the directory structure from the input.

### Usage

You can run `parser.py` in two different modes. Run these inside the `logs/` directory.

#### 1. Directory Parsing (Recommended)
You can target a parent directory representing an application (e.g. `tomcat/`). The script will recursively traverse the folder, find all `.json` files within its iteration subfolders, and write the output into the corresponding structure inside `parsed-logs/`.

```bash
python parser.py tomcat
```
*Outputs will be saved dynamically to `parsed-logs/tomcat/iterationX/parsed_...`*

#### 2. Single File Parsing
You can provide a specific json file you want to format:

```bash
python parser.py tomcat/iteration1/ministral-3_14b-cloud_2026-03-31_020039.json
```
*Output will directly be written into `parsed-logs/`*
